# 共通環境同期ワークフロー (sync-env)

このワークフローは、`env/` 内のテンプレートから必要なアセットを取り込み、プロジェクトに合わせて最適化（不要なものの削除）を行う「一方向の同期」プロセスを提供します。

## 手順

### 1. プロジェクト分析
- `env/README.md` を読み、提供されているアセットの全体像を把握してください。
- 現在のリポジトリの構造、主要言語を分析し、対象スタック（Android, TypeScript, Python）を特定してください。

### 2. テンプレートの展開とリネーム（Pull）
特定したスタックに基づき、アセットをプロジェクトルートに展開してください。

- **アンカー設定**: `env/antigravityrule.template` -> `./.antigravityrule`
- **AI ルール**: `env/agent_template/` -> `./.agent/`
- **GitHub**: `env/github_template/workflows/` -> `./.github/workflows/` (スタック別テンプレート含む)
- **Hooks/Scripts**: `env/hooks_template/` -> `./.hooks/`, `env/scripts_template/` -> `./scripts/`

**※ 展開時、必ずドット付きの正しい名前にリネームしてください。**

### 3. 展開後の最適化（Subtractions）
コピー完了後、直ちに「引き算」による最適化を実行してください。

1. **`.antigravityrule` のクリーンアップ**: 不要な技術スタックのインデックスと、`rules:` リスト内の不要なファイルパスを削除してください。
2. **ルールの整理**: `./.agent/rules/` 内の不要なスタック別ルールファイルを削除してください。

### 4. 完了報告とマニュアルフィードバックの案内
- 新たに展開・更新したファイルの一覧をユーザーに報告してください。
- **重要**: もしプロジェクト側で既存ルールへの「改善」が見つかった場合は、AI は勝手に同期元を更新せず、以下の手順をユーザーに提案してください。
    - 「この改善を共通基盤に反映するため、`antigravity-shared-env` リポジトリに Issue を作成しましょうか？」

### 5. 後処理
- 必要に応じて、`setup-labels` や `setup-hooks` スクリプトの実行を提案してください。
