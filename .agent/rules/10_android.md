# Android 開発プロトコル

Android 開発において、AI エージェントが自律的にプロジェクト構成を理解し、適切な品質管理を行うための手順です。

## 1. プロジェクト構成の自律探索
作業を開始する前に、必ず以下のファイルを確認し、プロジェクトの構造を特定してください。
- **ビルド・依存関係**: `build.gradle`, `build.gradle.kts`, `settings.gradle`, `settings.gradle.kts`, `gradle.properties`
- **マニフェスト**: `AndroidManifest.xml`
- **モジュール構造**: マルチモジュール構成か、単一モジュールか。

## 2. ツール設定の尊重
プロジェクト内に以下の設定ファイルが存在する場合、そこに記述されたルールや整形設定を最優先してください。
- **静的解析・整形**: `.editorconfig`, `ktlint`, `detekt` 設定、および Google Java Style などの特定設定。
- **ビルド構成**: 各モジュールの `build.gradle` に記載された SDK バージョンや、依存ライブラリのバージョン。

## 3. 実装後の検証プロトコル
コーディング完了後、ユーザーに進捗を報告または Pull Request を作成する前に、プロジェクト固有の検証タスクを特定して実行してください。

1. **検証タスクの特定**: `gradlew tasks` 等を実行し、ビルド（`assemble`系）、テスト（`test`系）、Linter/Formatter（`lint`, `ktlint`等）の適切なタスクを特定する。
2. **実行と修正**: 特定したタスクを実行し、すべてのエラーと警告を解消するまで繰り返す。
3. **報告**: PR 本文の「確認事項 (Verification)」には、**実際に実行したプロジェクト固有のコマンド**とその結果を記載する。

---

## 補足
- **一時ファイル**: 解析やビルド出力の一時的な加工が必要な場合は、リポジトリ直下の `temp/`（`.gitignore` で保護済み）を使用し、作業完了後に削除してください。
- **実行権限**: `gradlew` 等の実行コマンドの許可については `auto-commands.yaml` に従ってください。
