# Python 開発プロトコル

Python 開発において、AI エージェントが自律的にプロジェクト構成（パッケージ管理、スタイル、型ヒント）を理解し、適切な品質管理を行うための手順です。

## 1. プロジェクト構成の自律探索
作業を開始する前に、必ず以下のファイルを確認し、プロジェクトの構造を特定してください。
- **パッケージ・環境管理**: `pyproject.toml`, `requirements.txt`, `uv.lock`, `poetry.lock`, `environment.yml`, `Pipfile`
- **ツール構成**: `ruff.toml`, `.flake8`, `setup.cfg`, `tox.ini`

## 2. ツール設定の尊重
プロジェクト内に以下の設定ファイルが存在する場合、そこに記述されたルールを最優先してください。
- **Lint / Format / 型チェック**: `ruff`, `black`, `isort`, `mypy`, `pyright` 等の設定。
- **スクレイピング原則**: スクレイピングを行う場合は、対象サイトの `robots.txt` およびクロールレート制限を必ず確認し、負荷を制御すること。

## 3. 実装後の検証プロトコル
コーディング完了後、ユーザーに進捗を報告または Pull Request を作成する前に、プロジェクト固有の検証手順を実行してください。

1. **検証手段の特定**: `uv run ...`, `pytest`, `tox`, `make` 等、プロジェクトで標準的に使用されている検証コマンドを特定する。
2. **実行と修正**: 特定したコマンドを実行し、シンタックスエラー、実行時エラー、テスト失敗、Lint 警告を解消する。
3. **報告**: PR 本文の「確認事項 (Verification)」には、**実際に実行したプロジェクト固有のコマンド**とその結果を記載する。

---

## 補足
- **実行推奨**: 環境の整合性を保つため、プロジェクトが対応している場合は `uv run` を優先的に使用し、直接の `python` 実行は避けてください。
- **実行権限**: `uv`, `pytest` 等の実行コマンドの許可については `auto-commands.yaml` に従ってください。
