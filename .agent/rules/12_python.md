# Python / Scraping 規約 [12_python.md]

Python 開発、ウェブスクレイピング、およびスクレイピングによるデータ構築における統一ルールです。

## 1. コーディング規約 (Python)

- **スタイル指定**:
  - 原則として `PEP 8` に準拠した実装を行ってください。
  - `black` や `isort`, `flake8` が導入されている場合、それらを活用してコード品質を担保してください。
- **型ヒント**:
  - 可能な限り `typing` モジュールを使用した型ヒントを記述し、コードの可読性を高めてください。
- **仮想環境**:
  - ライブラリの導入は `venv` や `pipenv`, `poetry` 等の仮想環境管理ツールの指示に従ってください。

## 2. スクレイピング規約

- **robots.txt の尊重**:
  - 対象サイトの `robots.txt` を確認し、スクレイピングが禁止されていないか、またはクロールレートに制限がないか確認してください。
- ** polite スクレイピング**:
  - サーバーへの負荷を軽減するため、適切な `sleep` を挟み、一括リクエストを避けてください。
  - `User-Agent` を適切に設定してください。
- **エラーハンドリング**:
  - HTTP 4xx, 5xx やタイムアウトに対する適切なリトライ処理、例外処理を実装してください。

## 3. データベース構築・CI 連携

- **データ品質**:
  - スクレイピング後のデータ保存前にバリデーションを行い、不正なデータが DB に混入するのを防いでください。
- **CI 自動実行**:
  - GitHub Actions 等での定期実行を想定し、環境変数 (SECRETS) から API キーや認証情報を取得するように設計してください。

## 4. 許可されているコマンド

自動実行が許可されているコマンド（SafeToAutoRun: true）の最新リストについては、**`auto-commands.yaml` を参照・遵守**してください。

## 5. 実装完了後の必須検証

これらのチェック項目は **PR 本文の「確認事項 (Verification)」に記載** し、すべてパスしてからレビューを依頼してください。

1. [ ] シンタックスエラーおよび実行時エラーがないこと
2. [ ] ユニットテストがすべてパスすること
3. [ ] スクレイピングの対象サイトへの負荷が適切に制御されていること
4. [ ] DB 構築ロジックが正常に稼働し、意図したスキーマでデータが保存されること
