# GitHub Actions 開発プロトコル

GitHub Actions のワークフロー作成・修正において、AI エージェントが安全かつ効果的に CI/CD を構築するための手順です。

## 1. 既存ワークフローの自律探索
作業を開始する前に、必ず以下のディレクトリを確認し、現在の自動化状況を把握してください。
- **ワークフロー定義**: `.github/workflows/*.yml`, `.github/workflows/*.yaml`
- **アクション**: リポジトリ内にカスタムアクション (`action.yml`) があるか。

## 2. 実装プロトコル
1. **GitHub 機能を優先**: 可能であればマーケットプレイスの公式アクションや、プロジェクト内で既に安定して使われているアクションを再利用してください。
2. **セキュリティの考慮**: `secrets` の扱い、`presents` キーワードの使用、不要な権限の最小化を意識してください。
3. **環境変数の尊重**: プロジェクト固有の環境変数名や、既にある `env` 構成を崩さないように実装してください。

## 3. 実装後の検証プロトコル
1. **構文チェック**: YAML 構文エラーがないか確認する。可能であれば `actionlint` 等を用いてチェックを行う。
2. **ロジック確認**: ステップの順序、`if` 条件、`on` トリガーがユーザーの意図と合致しているか論理的に再確認する。
3. **報告**: PR 本文の「確認事項 (Verification)」には、**追加・変更したワークフローのトリガー条件と期待される動作**を明記する。

---

## 補足
- **検証ツール**: `act` などのローカル実行ツールが導入されているプロジェクトでは、積極的にそれらを使用してロジックを検証してください。
