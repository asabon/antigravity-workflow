---
description: 最後に保存された状態（Issue の栞）から作業を再開する
---

1. **ルールの再確認**: リポジトリ直下の `.antigravityrule` を読み込み、プロジェクト固有の制約とコミュニケーションルールを再確認する。
2. **全体像の把握**: `docs/99_progress/roadmap.md` を読み、現在のフェーズを確認する。
3. **着手タスク（Issue）の特定**:
   - 以下の優先順位で作業対象を特定します：
     1. **PR 紐付け**: `gh issue list --state open --json number,title,pullRequests` を実行。`pullRequests` が存在する Issue があれば、それを最優先候補とする。複数ある場合はユーザーに選択を求める。
     2. **ブランチ名**: PR がない場合、現在のブランチ名（`git branch --show-current`）に含まれる数字を Issue 番号とみなし、Open な Issue と照合する。
     3. **手動選択**: 上記で特定できない場合、`gh issue list --state open` で一覧を表示し、ユーザーに番号を選択してもらう。
4. **コンテキストの復元**: 
   - 特定された Issue の最新の `[Checkpoint]` コメント（あれば）と、Brain 側の `task.md` を読み込み、前回の中断地点と「次の一手」を把握する。
5. **ブランチの切り替え**: 
   - 継続作業の場合、Issue または PR に紐付いた作業ブランチを確認し、`git checkout <ブランチ名>` で切り替える。
6. **ユーザー報告**: 現在のフェーズ、タスクのステータス、完了事項、および次に着手する内容を要約して報告する。
