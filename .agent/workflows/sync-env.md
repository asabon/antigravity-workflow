# 手順

1. `env/README.md` または `env/index.md` 等の関連ファイルを読み込み、提供されている共通ルールの全体像を把握してください。
2. 現在のリポジトリの構造、主要言語、既存の `.antigravityrule` や `.github/` ディレクトリ、スクリプト類の内容を分析してください。
3. 分析結果に基づき、`env/` 内のアセット（AIのルール、GitHub Actions、Issue テンプレート、フック、スクリプトなど）の中で、どれを取り込むべきか、または既存のものとどうマージすべきか提案を作成し、ユーザーに確認を実施してください。
4. ユーザーから提案の承認を得たら、適切なファイルのコピー、マージ、または新規作成を実行してください。
    - 注意: 完全な上書きは既存の個別設定を壊す可能性があるため、安全なマージや適切な差分追加を優先してください。
