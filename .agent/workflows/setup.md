description: プロジェクトの初期設定を行い、不要なファイルを削除します
---
このコマンドは、テンプレートから新しく作成されたプロジェクトの初期セットアップを行うためのコマンドです。

AIエージェントは、以下の手順で現在のプロジェクトの初期化を行ってください。

1. **ロードマップの初期化**
   - `docs/99_progress/roadmap.md` の内容がテンプレートの初期コンテンツ（例: `Antigravity Shared Environment Roadmap` というタイトルが含まれている等）のままの場合、またはファイルが存在しない場合は、`.agent/templates/roadmap.md` をベースにして現在のプロジェクト用に新しく作成・上書きしてください。

2. **ルールの最適化（技術スタックの特定）**
   - まず、ディレクトリ内のファイル（`package.json`, `pyproject.toml`, `build.gradle` 等）を走査し、プロジェクトの技術スタックを特定してください。
   - **もしファイルが空、またはスタックを特定できない場合は、独断でルールを削除せず、必ずユーザーに使用する技術スタック（Python, Android, TypeScript 等）をヒアリングしてください。**
   - 特定あるいはユーザーからの回答を得たスタックに基づき、`.antigravityrule` の「技術スタック別ルール」セクションから該当しない項目を削除してください。
   - 同時に、不要と判断した `.agent/rules/` 以下のスタック別ルールファイルも削除してください。

3. **階層化構成の初期化**
   - プロジェクト固有のルールを記述するためのディレクトリ `.agent/custom/` を作成し、基盤リポジトリにあるテンプレートから `README.md` をコピー（または作成）してください。

4. **テンプレート固有ファイルのクリーンアップ**
   - テンプレートの説明がそのまま入っている `README.md` を、現在のプロジェクト向けにプレーンな状態に書き換えてください（現在のプロジェクト名と簡単な説明のみを記述した状態にしてください）。
   - テンプレートリポジトリに対する貢献ガイドである `CONTRIBUTING.md` を削除してください。
   - テンプレートリポジトリ固有のライセンスである `LICENSE` ファイルを削除してください。

4. **セットアップ完了の報告**
   - 以上の手順が完了したら、ユーザーに初期化が完了したことと、削除・変更したファイルの一覧を報告してください。
