---
description: 進捗を保存し、次回再開時のための「栞（しおり）」を GitHub Issue に残します
---

1. 現在のステータスを Brain 側の `task.md` に反映する。
2. **ブランチの確認**: `& git branch --show-current` を実行し、現在の作業ブランチ名を確認する。
3. **中断パターンの判定**:
   - `main` ブランチにいる場合、または全ての作業が完了している場合は **パターンB（一区切り）** として扱う。
   - 作業ブランチにいて、未完了のタスクがある場合は **パターンA（作業途中）** として扱う。
4. **進捗の可視化 (Checkpoint)**:
   - **パターンA (中断時)**: `task.md` の内容を要約（完了済みタスク、未解決の課題、次の一手）し、GitHub Issue に `**[Checkpoint: In Progress]**` としてコメントを投稿する。
   - **パターンB (一区切り)**: コメント投稿は行わず、Draft PR への push のみ実施する。
5. **Brain 側の更新**: 同様の情報を `task.md` に記録し、コンテキストを保存する。
6. ユーザーに現在の進捗と、次回の再開方法（`/resume`）を報告する。
