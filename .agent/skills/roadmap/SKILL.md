# roadmap Skill

## 目的 (Purpose)
プロジェクトの全体進捗を管理する `docs/status/roadmap.md` の更新操作をカプセル化し、AI による置換ミス（改行コードのズレなど）や一時ファイルの不正作成を防止します。

## 1. 運用ルール（ガードレール）
- **ステータス遷移**:
  - タスクは 原則として `[ ]` (未着手) -> `[/]` (着手中) -> `[x]` (完了) の順で遷移します。
- **Issue 連動**:
  - タスクを着手（`[/]`）にする際、または計画確定時に、必ず対応する **GitHub Issue 番号（例: `#61`）** をタスク説明の前に付与してください。
  - *正*: `- [/] #61 4-1. タスク名`

## 2. 付属スクリプトの使用方法
直接ファイルを編集する代わりに、以下の Python スクリプトを使用することで安全に更新できます。

### 📄 `update_roadmap.py`
- **概要**: `roadmap.md` 内の特定のタスク番号のステータスや Issue 番号を更新します。
- **実行方法**:
  ```bash
  python .agent/skills/roadmap/scripts/update_roadmap.py --task <タスク番号> --status <新ステータス> [--issue <Issue番号>]
  ```
- **引数**:
  - `--task`: `1-1`, `4-1` などのタスク番号（必須）
  - `--status`: `"[ ]"`, `"[/]"`, `"[x]"` などのステータス記号（必須）
  - `--issue`: `#123` などの Issue 番号（任意）
