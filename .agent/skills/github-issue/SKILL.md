## 1. Issue の作成
- **タイトルの検証**:
  - タイトルに `feat:`, `fix:` 等の接頭辞（プレフィックス）、カテゴリ、Issue 番号が**含まれていない**ことを確認します。
  - タイトルは**日本語**で記述されていることを確認します。
  - *正*: `割引計算ロジックの実装` / *否*: `feat: 割引計算ロジックの実装`

- **テンプレートの利用**:
  - `.github/issue_template/` 配下の適切なテンプレート（例：`task.md`）を使用します。

- **安全な作成**:
  - 長文になる場合は CLI エラー防止のため一時ファイルを作成し、`--body-file` オプションを使用します。
  - **一時ファイルの作成場所の優先順位** (直下作成厳禁):
    1. **Artifact ディレクトリ** (システム提供の `Artifact Directory Path` 絶対パス)
    2. **`temp/` ディレクトリ** (.gitignore で除外済み)
  - 作成後、必ず一時ファイルを削除します。

- **ラベルの付与**:
  - 作成時に必ず適切な **「分類ラベル」** を付与してください。
  - *代表例*: `enhancement` (機能追加), `bug` (バグ修正), `chore` (雑務), `documentation`
  - *詳細な一覧*: [03_github.md](../../rules/03_github.md) のセクション5 を参照してください。

---

## 2. Issue の更新（本文編集）
- **安全な編集**:
  - 本文を更新（Edit）する際は、長文になる場合に限り一時ファイルを生成し、`gh issue edit <Issue番号> --body-file <ファイル名>` で実行することを**強く推奨**します。
  - **一時ファイルの作成場所の優先順位** (直下作成厳禁):
    1. **Artifact ディレクトリ** (システム提供の `Artifact Directory Path` 絶対パス)
    2. **`temp/` ディレクトリ** (.gitignore で除外済み)
  - 実行後は必ず一時ファイルを削除します。

---

## 3. コメントの追加（栞/Checkpoint等）
- **コメント投稿**:
  - `gh issue comment <Issue番号> --body "内容"` を実行します。
  - 栞（Checkpoint）などの長文を投稿する際は、`--body-file` で一時ファイルを指定する方法が安全です。
