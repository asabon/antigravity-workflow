# 開発ワークフロー管理スキル (workflow)

AIのステータス（ACTIVE/IDLE）およびフェーズ誤認を物理的・システム的に防ぎ、開発サイクルを円滑に進めるための総合スキルです。

---

## 🧠 1. 状態遷移の自動演算 (`calc_next_state.py`)

エージェントは回答の末尾、コマンド完了時、フェーズ移行時を問わず、必ず本スクリプトを走らせて次の状態を演算しなければなりません。

- **モジュール名**: `scripts/calc_next_state.py`
- **使用例**:
  ```powershell
  python .agent/skills/workflow/scripts/calc_next_state.py --status ACTIVE --phase REVIEWING --action "/cleanup"
  ```
- **絶対ルール**: 演算スクリプトが算出した `status_bar` 文字列のみを SSOT（唯一の真実）として採用し、ステータスバーに記載すること。

---

## 🦾 2. 開発サイクルの実行自動化 (補足手順)

### 🚀 flow-kickoff (環境セットアップ)
タスク準備フェーズ（`PREPARING`）における初期調査・ブランチ作成等を着実に実行します。

1. **ロードマップの解読**: `docs/status/roadmap.md` から現在の対象タスクと依存関係を確認。
2. **Issue の特定と同期**: `gh issue list` 等で対象を確定し、ユーザーとの合意を Issue 本文へ同期。
3. **task.md の初期化**: Brain 側に作業ステップをブレイクダウンした上で、Issue へのCheckpointコメントを投稿。
4. **作業ブランチの作成と公開**:
   - 作成直前に必ず `verify-branch-name.ps1` でブランチ名を検証し、合格すること。
   - 初期コミット後、`gh pr create --draft` で作業を早期公開。
5. **ロードマップ更新**: 対象タスクを着手中（`[/]`）にマーキング。

### 🏁 flow-wrapup (事後検証・PR 最終化)
タスク完了フェーズ（`REVIEWING`）における最終監査と、確実なマージ・クリーンアップを実現します。

1. **最終動作検証**: すべてのテストやビルドのローカルパス確認。`git diff` でのセルフレビュー。
2. **PR 本文の最終チェック**: タイトルプレフィックスの禁止、`Closes #XX` の確認。
3. **ロードマップの完了更新**: 対象タスクを完了（`[x]`）マークし、コミットに含める。
4. **PR の Ready 化**: `gh pr ready` の実行（事前に `verify-pr.ps1` による物理検証をパスすること）。
5. **ユーザーへの修正報告**: `/cleanup` がいつでも実行可能であるアナウンスを含めた完了報告。

> [!TIP]
> より詳細な例外ガードレール（未マージPR時の処理等）や禁止事項については、常に最上位の `.antigravityrule` を最優先してください。
